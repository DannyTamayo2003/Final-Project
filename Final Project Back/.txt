per 1) sempre nel BE facciamo endpoint o fetch? e perche meglio uno che l altro?
Fetch nel BE
Il backend deve fare la chiamata HTTP verso API terza (es. Ticketmaster). Questo è il fetch server-to-server.

Endpoint nel BE
Il frontend non chiama direttamente la terza API: chiama una route tua, per esempio GET /api/external-events. Quell’endpoint interno esegue il fetch e restituisce dati puliti al frontend.

Quindi il flusso corretto è:
Frontend -> tuo endpoint BE -> fetch verso API terza -> risposta al frontend

Perché è meglio dell’alternativa (frontend diretto verso API terza):

1 Sicurezza: la API key resta nel backend, non finisce nel browser.
2 Controllo: puoi filtrare solo eventi coerenti col progetto (raduni, car meeting, area geografica).
3 Stabilità: se l’API terza cambia formato, sistemi solo nel backend.
4 Uniformità: il frontend riceve sempre lo stesso formato dati.
5 Governance: gestisci limiti, timeout, errori, retry in un solo punto.
